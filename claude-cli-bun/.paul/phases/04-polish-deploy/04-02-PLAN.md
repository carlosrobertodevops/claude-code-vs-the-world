## AC-1: Seed popula todas as tabelas do spec

```gherkin
Given banco com schema aplicado mas vazio
When executa `bun run db:seed`
Then tabelas têm:
  - 3 usuários: admin@lavaflow.com (MANAGER), joao@lavaflow.com, maria@lavaflow.com (EMPLOYEE) — senhas bcrypt 12 rounds
  - 10 clientes, cada um com 1–3 veículos
  - 20 produtos (variedade de stock_qty incluindo baixo)
  - 5 tipos de serviço (com preços base)
  - 5 orçamentos (status variados)
  - 3 contratos (1 SIGNED, 2 PENDING)
  - 8 ordens de serviço em status variados
  - 3 entradas de fila no slug `lavaflow-centro`
  - 1 CarWashConfig slug=lavaflow-centro
  And re-executar seed é idempotente (truncate-and-reload)
```

## AC-2: Verify exercita fluxos críticos

```gherkin
Given seed aplicado e app rodando (localhost:3000) OU db direto via repo
When executa `bun run verify`
Then verify:
  1. Chama POST /api/auth/login com admin e recebe JWT (ou valida via repo direto)
  2. Cria produto via POST /api/inventario/produtos
  3. Cria OS via POST /api/servicos
  4. Chama GET /api/fila/publica/lavaflow-centro sem auth e recebe array
  5. Chama GET /api/orcamentos/:id/pdf e recebe Blob com content-type application/pdf
  And exit code 0 se tudo passa, !=0 com log claro se falha
```
