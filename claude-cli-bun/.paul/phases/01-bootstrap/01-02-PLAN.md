<objective>
## Goal

Configurar as primitivas de configuração do projeto: `drizzle.config.ts`, arquivos de env (`.env` + `.env.example`), ajustes finos em `tsconfig.json` e `.gitignore`. Este plano é a ponte entre scaffold (01-01) e as camadas de domínio/schema (Phase 2).

## Purpose

Sem `drizzle.config.ts`, `agent-schema` (Phase 2) não consegue rodar `db:generate`/`db:migrate`. Sem `.env` populado, o app não sobe. Sem `.env.example` versionado, onboarding quebra.

## Output

- `drizzle.config.ts` apontando para `drizzle/schema/` e usando `DATABASE_URL`
- `.env` (local) e `.env.example` (versionado) com todas as variáveis do LavaFlow
- `.gitignore` cobrindo `.env*` (exceto `.env.example`), `node_modules`, `.next`
- `tsconfig.json` com path aliases adicionais e `moduleResolution` consistente com Bun
</objective>

<task type="auto">
  <name>Task 1: Criar drizzle.config.ts</name>
  <files>drizzle.config.ts</files>
  <action>
    Criar `drizzle.config.ts` na raiz:
    ```ts
    import { defineConfig } from "drizzle-kit";

    export default defineConfig({
      schema: "./drizzle/schema/index.ts",
      out: "./drizzle/migrations",
      dialect: "postgresql",
      dbCredentials: {
        url: process.env.DATABASE_URL!,
      },
      strict: true,
      verbose: true,
    });
    ```

    Avoid:
    - Hardcoded connection string.
    - Dialect "sqlite" ou "mysql" — spec trava em PostgreSQL.
    - Apontar `schema` para arquivo único — tem que ser `./drizzle/schema/index.ts` (agent-schema re-exporta).
  </action>
  <verify>`test -f drizzle.config.ts` E `bunx drizzle-kit --help` não falha com erro de config</verify>
  <done>AC-1 satisfied: drizzle.config.ts válido e apontando para pastas corretas</done>
</task>

<task type="auto">
  <name>Task 3: Ajustar tsconfig.json e .gitignore</name>
  <files>tsconfig.json, .gitignore</files>
  <action>
    Em `tsconfig.json`, garantir (manter o que o scaffold colocou, só adicionar):
    - `"strict": true`
    - `"noUncheckedIndexedAccess": true`
    - `"moduleResolution": "Bundler"` (ou "NodeNext" se gerado assim)

    ```

    .env.*.local
    !.env.example

    drizzle/.drizzle/

    out/

    # node
    node_modules/

    # os
    .DS_Store
    ```

    Nota: `bun.lockb` DEVE ser commitado normalmente — remover do .gitignore se scaffold adicionou. (reconfirmar: bun.lockb é binário mas DEVE ser versionado como lockfile; remover `bun.lockb` do gitignore.)

    Avoid:
